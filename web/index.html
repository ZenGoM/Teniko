解説: なぜPDF変換が必要なのか？

⚠️ Wangアノテーションの危険性

Wangアノテーション（コダック・アノテーション）を現代のシステムでそのまま保持・展開することは、セキュリティやガバナンスの観点から非常に大きなリスクを伴います。

  • 個人情報（PII）の漏洩: マスキング（黒塗り）が指示データとしてのみ保存されているため、現代のビューアで開くと隠したはずの機密データが露出してしまいます。
  • 法的証跡の喪失: 独自のプライベートタグを使用しているため、一般的なビューアではスタンプや署名が「不可視」になり、証跡としての有効性が失われます。
  • システムクラッシュ: 不良なメタデータタグが書き込まれていることが多く、現代の画像処理ライブラリで致命的なエラーを引き起こす原因になります。

✨ PDF/A変換のメリット

標準的なPDF/A形式へ変換（トランスコーディング）することで、データの価値を維持・向上させます。

  • 長期保存規格への適合: 標準的な注釈オブジェクトとして精密に変換し、将来にわたって確実に表示・閲覧できるようになります。法的証跡も復元されます。
  • コラボレーション価値の維持: バーンイン（画像への焼き付け）とは異なり、独立したベクトルオブジェクトとして保持されるため、テキスト抽出や検索への活用が可能です。
  • リスクの排除: 不良タグを取り除きクリーンなPDFへ再構築。マスキングも標準機能として安全に再構築されるため、機密データの露出を防ぎます。

ℹ️ 変換時の損失・変容（デグレード）

独自フォーマットと現代の規格の差異により、一部の装飾やメタデータは以下の通り変容または失われます。

オブジェクト別の変容

  • 直線・図形: 破線や角の接合仕様が失われ実線へ。特殊な塗りつぶしもベタ塗りへ縮退。
  • 曲面図形: 楕円パラメータは失われ、多角形（ポリライン）として近似描画。
  • テキスト・付箋: 影などの装飾が破棄。フォントのマッピングにより表示位置のズレが生じる場合があります。
  • 手書き線: 滑らかな補間が引き継がれず角張って表示される場合があります。

メタデータの損失

  • 編集可否フラグ: 移動・回転・リサイズ可能といった属性は失われ、デフォルト値にリセットされます。
  • 識別子と履歴: 固有識別子（Guid）や変更ユーザー名（ModifiedBy）のデータは失われます。
  • 日時の上書き: 変更日時は保持されず、作成日時で強制的に上書きされます。

使い方 (Quick Start)

以下のコマンドを実行するだけで、TIFFファイルがPDFに変換されます。

.\teniko.exe <入力TIFFファイルのパス> <出力PDFファイルのパス>

実行例:

.\bin\Debug\net8.0\teniko.exe sample.tif output.pdf

※開発環境での実行:

dotnet run -- sample.tif output.pdf
