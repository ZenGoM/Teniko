# Teniko テスト仕様書

本ドキュメントは、TIFF to PDF Converter「Teniko」の品質を担保するために実施するテストの仕様を定めたものです。

## 1. 起動・ユーザーインターフェーステスト

本アプリケーションがユーザーからの入力やファイル指定を正しく受け付け、適切にフィードバック（エラーや処理完了の表示）を返すことを確認します。

| テストID | テスト項目 | 手順 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| UI-01 | 引数なし起動テスト | コマンドラインやスタートメニューから引数なしで `teniko.exe` を起動する。 | 使い方と「ドラッグ＆ドロップして [Enter] を押してください」というプロンプトが表示され、入力待機状態になること。 |
| UI-02 | コンソールへのドラッグ＆ドロップテスト | UI-01の待機画面で、TIFFファイルをコンソールウィンドウにドラッグ＆ドロップしてEnterキーを押す。 | ファイルパスが認識され、入力ファイルと同じフォルダに `.pdf` 拡張子でPDFが出力されること。処理終了後にEnter待機画面になること。 |
| UI-03 | 引数1つ指定での起動テスト | `teniko.exe <TIFFファイルパス>` のように引数を1つ指定して実行する（ファイルをexeにドラッグ＆ドロップする操作も含む）。 | プロンプトを出さずに変換処理が開始され、入力ファイルと同じフォルダに `.pdf` 拡張子でPDFが出力されること。処理終了後にEnter待機画面になること。 |
| UI-04 | 引数2つ指定での起動テスト | `teniko.exe <TIFFファイルパス> <出力PDFパス>` を指定して実行する。 | 指定した出力PDFパスにファイルが正しく保存されること。処理終了後にEnter待機画面になること。 |
| UI-05 | 不正ファイル指定時のエラー処理 | 存在しないTIFFファイルのパスを指定して実行する。 | 「エラー: 入力ファイルが見つかりません」といった適切なエラーメッセージが表示され、即座にウィンドウが閉じずにEnter待機状態になること。 |
| UI-06 | 処理完了後の待機確認（MS Store審査対策） | 正常な変換処理、あるいはエラー発生後、画面が即座に閉じないか確認する。 | 必ず `[Enter]キーを押して終了します... (Press Enter to exit...)` と表示されて待機し、Enterキーを押すことで初めてウィンドウが閉じること。 |
| UI-07 | 出力先ファイルのロック競合テスト | 変換先のPDFファイル（例: `output.pdf`）をAdobe Acrobat等で開いたまま、同名で上書き出力しようとする。 | ファイルアクセス権限エラーが `try-catch` で捕捉され、「処理中に致命的なエラーが発生しました」と表示された後、強制終了せずにEnter待機状態になること。 |

## 2. TIFFファイル読み込み・解析テスト

TIFFファイルの各種フォーマットや、Wangアノテーション（Tag 32932）が含まれる場合の読み込み挙動を確認します。

| テストID | テスト項目 | 手順 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| Read-01 | マルチページTIFFの読み込み | 複数ページからなるTIFFファイルを入力する。 | 全てのページ数、画像サイズ（Width/Height）、解像度（DPI）が正しく認識され抽出されること。 |
| Read-02 | アノテーション無しTIFFの読み込み | Wangアノテーション(Tag 32932)を持たないTIFFファイルを入力する。 | エラーで停止することなく、純粋な画像のみのPDFとして正常に変換・出力されること。 |
| Read-03 | Wangアノテーション付きTIFFの読み込み | Wangアノテーションが付与されたTIFFファイルを入力する。 | Tag 32932のバイナリデータが検出され、ページごとにアノテーションデータとして保持されること（コンソールに検出ログが出力されること）。 |
| Read-04 | 破損したTIFFファイルの読み込み | 画像データが破損している、またはTIFFとして不正なファイルを入力する。 | `try-catch` によって例外が捕捉され、「処理中に致命的なエラーが発生しました」というメッセージとともに安全に処理が中断されること。 |
| Read-05 | 大容量・多ページTIFFの処理テスト | 数十〜数百ページに及ぶサイズの大きいマルチページTIFFファイルを変換する。 | メモリ不足（OOM）でクラッシュせず、正常に全ページがPDF化されること。 |
| Read-06 | 複数ピクセルフォーマットの互換性 | モノクロ2値（1bpp）、グレースケール（8bpp）、フルカラー（24bpp）など、異なる色深度のTIFFファイルを変換する。 | どのフォーマットであっても色が反転したり乱れたりせず、正しくPDF上に画像が描画されること。 |

## 3. Wangアノテーションパース・描画テスト

Wangアノテーションの各要素が、出力されるPDF上でPDF標準の注釈（Annotation）または図形として正しくマッピング・描画されるかを確認します。

| テストID | テスト項目 | 手順 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| Annot-01 | 直線（LineAnnot）の描画 | 直線アノテーションが含まれるTIFFを変換する。 | PDF上で、元の座標・太さ・色を維持した線分（Line Annotation）として描画されること。 |
| Annot-02 | テキスト（TextAnnot）の描画 | テキストアノテーションが含まれるTIFFを変換する。 | PDF上で、正しいテキスト内容、色でFreeText注釈として配置されること。フォントサイズや位置が元の画像に沿って適切にスケーリングされること。 |
| Annot-03 | ハイライト（FreeHandHighligtherAnnot） | ハイライトが含まれるTIFFを変換する。 | PDF上で、指定された色と領域でHighlight注釈（QuadPoints）として配置されること。 |
| Annot-04 | 付箋（StickyNoteAnnot）の描画 | 付箋アノテーションが含まれるTIFFを変換する。 | PDF上で、コメント（Text Annotation / Icon: Comment）として配置され、テキスト内容や指定色（前景色・背景色）が保持されていること。 |
| Annot-05 | 矩形（RectangleAnnot）の描画 | 矩形アノテーションが含まれるTIFFを変換する。 | PDF上で、Square注釈として配置され、線の太さや枠線の色などが再現されること。 |
| Annot-06 | スタンプ（RubberStampAnnot）の描画 | スタンプアノテーションが含まれるTIFFを変換する。 | 現在の仕様に沿って、テキストアノテーションとして代用・描画されること。 |
| Annot-07 | 解像度（DPI）に応じたスケーリング | DPI設定が異なる（例: 200dpi, 300dpi）TIFF画像に付与されたアノテーションを変換する。 | ピクセル座標系からPDFのポイント座標系（72dpi基準）へのスケール変換が正確に行われ、画像上の正しい位置にアノテーションが重なること。 |
| Annot-08 | 日本語テキストの文字化け確認 | Wangアノテーションのテキスト（TextAnnot / NoteAnnot）に、日本語（全角文字）が含まれるTIFFファイルを変換する。 | Shift-JIS (CodePage 932) として正しくデコードされ、PDF上で文字化けせずに日本語が表示されること。 |

## 4. PDF出力テスト

生成されたPDFファイルが各種リーダーで問題なく閲覧できるか、また画像の画質が維持されているかを確認します。

| テストID | テスト項目 | 手順 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| PDF-01 | PDFリーダー互換性テスト | 出力されたPDFを、Adobe Acrobat Reader, Microsoft Edge, Google Chrome 等の主要リーダーで開く。 | 問題なく閲覧でき、ファイル破損のエラー等が発生しないこと。 |
| PDF-02 | ページサイズ・向きの保持 | 縦長、横長など異なるサイズのページが混在するTIFFを変換する。 | 出力PDFの各ページが、元のTIFFのサイズ・アスペクト比を正確に反映していること。 |
| PDF-03 | 画質確認 | 出力されたPDFのベース画像（TIFFから変換された画像）を目視確認する。 | 画像が極端に劣化したり、欠損したりせずに表示されること（現状はPNGストリーム経由で埋め込まれる仕様）。 |
| PDF-04 | ページ順序の維持確認 | ページごとに異なる内容を持つマルチページTIFFを変換する。 | 出力されたPDFのページ順序が、元のTIFFのページ順序と完全に一致していること。 |

## 5. MSIXパッケージ・デプロイテスト

Microsoft Store向けのパッケージングおよびインストールに関するテストです。

| テストID | テスト項目 | 手順 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| Deploy-01 | MSIXビルドスクリプトの実行 | `build_msix.ps1` を実行する。 | エラーなくコンパイル、マニフェスト生成、署名が行われ、`Teniko.msix` パッケージが生成されること。 |
| Deploy-02 | インストールと起動 | 生成された `Teniko.msix` をWindows環境にインストールし、スタートメニューから実行する。 | 正常にインストールされ、スタートメニューのアイコンからコンソールアプリとして起動できること（UI-01の画面になること）。 |
