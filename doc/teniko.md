「Wangアノテーション（コダック・アノテーション）」の技術的な詳細情報（仕様）は、現在も入手自体は可能ですが、「公式ドキュメントとしてはほぼ絶滅状態にあり、現在はオープンソースの解析コードや、サードパーティの互換資料から仕様を紐解くのが主流」という、やや難易度の高い状態になっています。

開発言語（C#など）でこのデータをバイナリ解析・デバッグ、あるいはデータ移行（モダンなPDF注釈への変換）を試みられる開発者の方に向けて、現在手に入る情報の種類と、仕様のコアとなる構造をまとめました。

---

## 1. どこから詳細仕様（ソースコード・ドキュメント）を入手できるか？

オリジナルであるWang社やコダック社の公式仕様書（PDF等）は、現在公式配布されていません。しかし、以下のリソースから「完全に逆コンパイル・解析されたデータ構造」を入手できます。

### ① オープンソースのコンバータープロジェクト（GitHub）

文書管理の専門企業であるOrpalis社（現Acreat社グループ）が、レガシーなWangアノテーションをモダンなデータに変換するためのC# .NET用オープンソースプロジェクト「WangTagReading」をGitHub上に公開しています。

* **GitHubリポジトリ:** `Orpalis/WangTagReading`
* **得られる情報：** どのバイトがテキスト、色、座標を示しているかのシリアライズ／デシリアライズのロジックがC#コードとして完全に記述されているため、仕様書代わりにコードを読むのが最も確実です。

### ② アーカイブ・検証サイトの技術情報

ファイルフォーマットの保存を行っている有志コミュニティ「Just Solve the File Format Problem」のWikiページなどに、タグ番号などのメタデータ構造が保管されています。

* **キーワード:** `TIFF annotation data - Just Solve the File Format Problem`

## 2. 開発に役立つ「Wangアノテーション」の構造詳細

バイナリデータを直接覗く、あるいはプログラムでパースする場合、以下の仕様がベースとなります。

### ① ターゲットとなるTIFFタグ

Wangアノテーションのデータは、TIFFの「プライベートタグ」と呼ばれる領域にバイナリの塊（BLOBデータ）として丸ごと格納されています。

* **Tag ID:** `32932` (十六進数: `0x80A4`)
* **タグ名:** `Wang Annotation Data` または `eiStream Annotation Data`
* **データ型:** `UNDEFINED` または `BYTE`

### ② 格納されているデータの正体（ベクトルデータ構造）

タグ`32932`の中身は画像ではなく、以下のような**オブジェクト指向のベクトルグラフィックスデータ**がシリアライズされた、独自のバイナリパケット構造をしています。

主要なアノテーションタイプと、対応する内部データ構造は以下の通りです。

| アノテーション名（表示） | 内部データの内容 |
| --- | --- |
| **Typed Text（テキスト入力）** | 文字列データ（ASCII/Unicode）、フォント名、サイズ、配置座標（X, Y）、文字色 |
| **Attach-a-note（付箋）** | 背景色（多くは黄色）、枠線フラグ、内部のテキスト文字列、矩形範囲 |
| **Highlighter（蛍光ペン）** | 半透明（アルファチャンネル効果）の塗りつぶし矩形データ、色情報 |
| **Straight / Freehand Line** | 始点・終点の座標、または連続する点（Point型配列）の座標データ、線の太さ・色 |
| **Hollow / Filled Rectangle** | 矩形（Rectangle）の4頂点座標、枠線のみか・塗りつぶしかのフラグ、色 |

## 3. 自作プログラム等で扱う場合の注意点

もしC#や.NET等の環境でこれらを解析・PDFへ移行するプログラムを組まれる場合、以下の仕様の癖（バグの原因になりやすいポイント）に注意が必要です。

* **座標系の違い：** 当時のImaging for Windowsは、画像のピクセル座標（解像度 DPI に依存）をベースにアノテーションの配置位置を記録しています。PDFに変換する際は、PDF標準の座標単位（ポイント：1/72インチ）への計算・変換が必要です。
* **回転（Rotation）の不整合：** 画像自体が90度回転している（TIFFの`Orientation`タグが1以外）場合、Wangアノテーションの座標データが「回転前の生画像」を基準にしているケースがあり、表示位置がズレるバグが起きやすいです。
* **テキストの文字化け：** Windows 95〜98時代の日本語テキスト（Shift-JIS）でエンコードされている可能性が高いため、現代の環境（UTF-8やUTF-16）でデコードする際は適切なコードページ（CP932）を指定する必要があります。

完全に1からバイナリパーサーを作るのは骨が折れる仕様ですが、前述のGitHubのC#ソースコード（`WangTagReading`）をクローンして構造をトレースするのが、現在もっとも現実的かつ詳細な仕様の入手方法と言えます。

## 4. 現在のプログラムの仕様

本プログラム（Teniko）では、Wangアノテーションを含むTIFFファイルからPDFを生成するにあたり、以下の仕様で動作します。

### アノテーションのPDF注釈化（表示/非表示コントロール）
Wangアノテーションは画像に直接「焼き付け（フラット化）」されるのではなく、**PDF標準の「注釈（Annotations）」として付加**されます。
これにより、出力されたPDFファイルをAcrobat Reader等のPDFビュワーで開いた際、「注釈パネル」からテキストやハイライト等の**表示・非表示を個別に切り替えたり、削除したりすることが可能**です。
*   テキストや付箋（Note）は `PdfTextAnnotation`（コメントアイコン等）に変換されます。
*   直線、ハイライト、矩形などの図形アノテーションも、各々PDFの対応する注釈図形（Line, Highlight, Square）として出力されます。

### デバッグログの出力
TIFFファイルの仕様上、保存ソフトによってはWangアノテーションデータが消失しやすいという問題があります。
本プログラムは実行時、コンソール画面に以下のデバッグログを出力し、データの有無を可視化します。
*   `[Debug] Page X: WangAnnotationDataを検出しました (〇〇 バイト)。`
*   `[Debug] Page X: WangAnnotationData (タグ32932) は見つかりませんでした。`

もしアノテーションが変換されない場合は、このログを確認することで「元ファイルにデータが存在しないのか」を切り分けることができます。

## 5. ビルドと実行手順

本プロジェクト(`teniko`)をビルドして実行するための手順は以下の通りです。

### 前提条件
- .NET 8.0 SDK がインストールされていること。

### ビルド手順
1. コマンドプロンプトまたは PowerShell を開きます。
2. 本プロジェクトのディレクトリ（`Teniko.csproj` があるフォルダ）に移動します。
   ```powershell
   cd <プロジェクトフォルダのパス>
   # 例: cd C:\Users\INO\source\repos\teniko
   ```
3. 必要なライブラリを取得します（パッケージの復元）。
   ```powershell
   dotnet restore
   ```
4. 以下のコマンドを実行してビルドします。
   ```powershell
   dotnet build
   ```
5. ビルドが成功すると、`bin\Debug\net8.0\teniko.exe` が生成されます。

### 実行方法
プロジェクトディレクトリにて、以下の形式でコマンドを実行します。
```powershell
.\bin\Debug\net8.0\teniko.exe <入力TIFFファイルのパス> <出力PDFファイルのパス>
```
※または、`dotnet run -- <入力TIFFファイルのパス> <出力PDFファイルのパス>` でも実行可能です。
